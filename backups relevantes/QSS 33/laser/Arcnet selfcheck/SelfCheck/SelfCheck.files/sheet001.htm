Self-Diagnostic Program Online Manual
The Self-Diagnostic program is a program that enables self-diagnostics for ARCNET communication with Noritsu systems
Operation is simple, it will detect the abnormality as much as possible, (Operations are simple, it has the ability to detect most communication abnormalities.)
1. How to Use 
Select the Model, the Optional device then click START
to start the diagnosis.
Then follow the instructions shown on the screen.
The result will be shown upon completion of the diagnosis.  
Note: In order to change an environmental setting, such as a display language, select the Environment tab.
Note: For details of each diagnostic item, refer to the corresponding screens explained below.
2. Explanation of Screen
Self-diagnostic program consists of the following major screens: Initial, Environment and Diagnostic Items
Below is the description of each screen. 
2.1 Initial Screen
Below is the Initial screen.
Model: Select the model to diagnose.
ARCNET: Select to perform the ARCNET diagnostics.
Environment: Select the display language and so on.
START button: Click this button to start the ARCNET diagnostics.
STOP button: Click this button to stop the ARCNET diagnostics.
? (Help) button: Click this button to display the online help.
Endless check box:  Click this box to diagnose without time restriction.
Diagnostic Time: Specify the diagnostic time. (minutes)
Note: When Endless is selected, it is not allowed to specify the diagnostic time. 
2.2 Environment Screen
Below is the Environment screen.
LANGUAGE: Select the display language.
EN = English
JP = Japanese
Note: Changing the language will not affect the basic part of the screen, such as Menu and the selection items. 
Note: English is selected by default.
2.3 ARCNET Diagnostic Screen
This section describes the structure of ARCNET diagnostic screen. 
ARCNET diagnostic screen consists of the two screens - proceeding and result.
2.3.1 Diagnostic Screen
The following screen is shown while diagnostic is proceeding. 
START button: Click this button to start the ARCNET diagnostics.
STOP button: Click this button to stop the ARCNET diagnostics.
Display form: Select the display form from List and Node Map.
List: Shows the status of the ARCNET connection devices listed in the selected model. 
Normally select this display form.
Node Map: Display all of the node.
This display form is for test purpose, and is not used to diagnose.  
Availability of optional device: Click the check box when the displayed optional device is connected. 
Unknown： When the ARCNET connection device not listed and the selected model is detected, its node ID is displayed.  
Note: In this case, the node ID will be displayed in black letter with red background. 
Device status: Shows the diagnostics status of the ARCNET connection device. 
Descriptions are below: 
Dev. Name： Displays the name of the ARCNET connection device. 
NID： Displays the node ID of the ARCNET connection device. 
Note: Node ID: Node means ARCNET connection device and node ID is ID assigned by each device.
Connect： Displays connection status.
Once the diagnostic program detects a node ID, "*" is shown. 
The numeric value in the parentheses shows the number of times after detecting the node ID by the diagnostic program to　become undetected.
TX Err： Displays the number of sending transfer errors when the diagnostic program is communicated to the ARCNET connection device. 
Error/Count： Displays the Number of detected errors/Number of total checks of the diagnostic program.
OK ： The device communicates correctly.
− ： There is an abnormality in communication. 
Note: Abnormal device is specified by the diagnostic result.
Even if displayed that it is abnormal, the device is not abnormal.
RCN： Displays "Number of detected Reconfiguration".
Reconfiguration: This mean that the another ARCNET connection device joined to the communication or it pull out of the communication.
MYRCN： Displays "Number of detected My Reconfiguration".
My Reconfiguration: This mean that the subjected ARCNET connection device itself joined to the communication or it pull out of the communication.
Exist： Displays the detection status of connected device.
Note: N/A (not applicable)
Colors: Displayed color is explained.
　 Change ： Connected device being diagnosed or changing its status. 
　 Unknown Node ： This background color shows unknown device when detecting the ARCNET connection device that is not existed in the model.
　 Node Change ： Connected device whose connection status has been changed. 
　 Error Dev. ： Not used when diagnosing.
　 Error Cause ： Not used when diagnosing.
2.3.2 Diagnostic Result Screen
The following screen displays the diagnostic result. 
The description varies depending on the result. 
If an abnormality is found, confirmation items and corrective action are displayed.
OK button: After confirming each item, press this button to close the screen.
The following screen displays the details of the diagnostic result. 
When the main and accessories are errors, the main may be influenced by the error of accessories.
Perform the diagnosis from accessories.
START button: Click this button to start the ARCNET diagnostics.
STOP button: Click this button to stop the ARCNET diagnostics.
Colors: Displayed color is explained.
　 Change ： Not used for diagnostic result.
　 Unknown Node ： Not used for diagnostic result.
　 Node Change ： Not used for diagnostic result.
　 Error Dev. ： Shows the device in which the error occurs. 
　 Error Cause ： Shows the cause of the error of the device. 
Others:  Refer to 2.3.1 Diagnostic screen.
